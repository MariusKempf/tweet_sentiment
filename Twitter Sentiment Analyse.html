1. Einführung¶

Das Internet ist zu einer Plattform geworden, die von den meisten Menschen tglich ber einen lngeren Zeitraum genutzt wird. Auf Webseiten, sozialen Netzwerken, Blogs, Online-Shops, Bewertungsportalen und vielem mehr entstehen tglich groe Mengen an Daten in Form von geschriebenem Text. Diese von realen Personen stammenden Texte enthalten Informationen ber deren Meinung bzw. Stimmung gegenber einer bestimmten Sache. Dies knnen unter anderem Meinungen ber Unternehmen, Politiker, Prominente, Bcher, Hotels, Ereignisse oder neue Technologien sein. Um jedoch groe Datenmengen zu analysieren, bedarf es Techniken, die eine Analyse automatisiert durchfhren knnen. Der Bereich zur Analyse von Daten in Textform wird als Text Mining bezeichnet. Die automatisierte Auswertung von Texten mit dem Ziel, eine geuerte Haltung (das Sentiment) als positiv oder negativ zu erkennen wird Sentiment Analyse genannt.
Die Sentiment Analyse arbeitet mit Methoden aus der Statistik, dem Maschinellen Lernen und dem "Natural language processing. Die Arbeit von Medhat, Hassan und Korashy (2014) bietet eine bersicht von Methoden der Sentiment Analyse. Diese sind in der folgenden Abbildung dargestellt.





Ziel der vorliegenden Arbeit ist es, ein Modell zu entwickeln um Tweets hinsichtlich der Stimmungslage des Autors gegenber einem bestimmten Thema zu klassifizieren. Dazu wird zunchst ein lexikonbasierter Ansatz als Baseline-Modell verwendet. Anschlieend wird versucht, mit neuronalen Netzen die Genauigkeit des Baseline-Modells mglichst weit zu bertreffen. Die entwickelten Modelle werden im Anschluss auf die gesammelten Tweets angewendet. Dazu wurden im Zeitraum vom 13.11.2018 bis einschlielich 31.01.2019 alle englischsprachigen Tweets, die den Hashtag #apple oder #tesla enthalten, gesammelt. Dadurch lsst sich ein zeitlicher Verlauf des Stimmungsbildes zum jeweiligen Thema ableiten. Ziel dabei ist es, Aufflligkeiten im Stimmungsverlauf zu identifizieren und mit mglichen unternehmensbezogenen Ereignissen in Verbindung zu bringen.

def cleanDataTrain(data):
    tweets = []
    labels = []
    data['label'].replace([4], 1, inplace = True)
    for i in range(len(data.text)):
        if (i + 1) % 100000 == 0:
            print("{} von {} Tweets bearbeitet.".format(i + 1, len(data.text)))
        #Durchfhrung der oben aufgelisteten Bearbeitungsschritte
        tweet = data.text[i]
        label = data.label[i]
        tweet = tweet.lower() 
        tweet = re.sub(r"@\S+", "", tweet)
        tweet = re.sub(r"http\S+", "", tweet)
        tweet = re.sub(r"www.\S+", "", tweet)
        tweet = " ".join([contractions_mapping.get(i, i) for i in tweet.split()])
        tweet = re.sub("[^a-zA-Z]", " ", tweet)
        words = nltk.word_tokenize(tweet)
        tweet = []
        for w in words:
            tweet.append(w)
        tweet = " ".join(tweet)
        tweets.append(tweet)
        labels.append(label)   
    data = pd.DataFrame({'label': labels, 'text': tweets})    
    return data

3.1 - Baseline Modell VADER Sentiment¶

Als Baseline Modell wurde das Sentiment Analyse Tool VADER (Valence Aware Dictionary and sEntiment Reasoner) verwendet. Der Entscheidung VADER Sentiment zu verwenden liegt zum einen das Paper von Hutto und Gilbert (2014) zugrunde. Die Autoren haben VADER mit unterschiedlichen Tools verglichen und festgestellt, dass VADER speziell fr "Micro-Blogging" sehr gut geeignet ist. Zum anderen wurde es auch fr die vorliegende Arbeit mit anderen Tools verglichen und konnte dabei am besten abschneiden.
Bei VADER Sentiment handelt es sich um ein regel- und lexikonbasiertes Tool, welches speziell fr in Social Media geuerte Gefhle ausgelegt ist und sich in diesem Kontext beweisen konnte. Da es auf einem Lexikon basiert, werden keine Trainingsdaten bentigt und VADER kann direkt zur Klassifizierung verwendet werden. Angewendet auf einen Text, im vorliegenden Fall auf einen Tweet, bestimmt VADER nicht nur ob dieser positiv oder negativ ist, sondern gibt auch an in welchem Ma dieser positiv bzw. negativ ist.

Wird VADER beispielsweise auf den Text "Vader is a super cool tool!" angewendet, ergeben sich folgende Werte:

Sentiment Metric Score
Positiv 68,40%
Neutral 31,60%
Negativ 0,0 %
Compound 0,7574

Die Werte zu positiv, neutral und negativ geben die prozentuale Verteilung der enthaltenen Wrter (unter der Bercksichtigung zustzlicher Regeln) in die Klassen an. Der Wert Compound entspricht der Summe aller Lexikonbewertungen der Wrter, normiert auf den Bereich -1 (extrem negativ) und +1 (extrem positiv). Dieser Compound Score wird fr die Klassifizierung verwendet. bliche Schwellenwerte fr die Entscheidung ob ein Tweet positiv, neutral oder negativ ist, sind:

  • Positiv: compound score > = 0,05
  • Neutral: -0,05 < compound score > 0,05
  • Negativ: compound score < = 0,05

Da in dieser Arbeit allerdings lediglich in positiv oder negativ klassifiziert wird, wurde 0.0 als Schwellenwert festgelegt. Neutrale Tweets mit einem Wert von 0.0 werden als positiv klassifiziert.

3.2 - Word2Vec¶

Word2Vec ist ein rechnerisch effizientes Vorhersagemodell fr das Lernen von Vektoren aus rohem Text. Dabei handelt es sich um unberwachtes Lernen (engl. unsupervised learning). Somit wird das Reprsentationsmodell ohne ueren Input bezglich eines Ziels generiert. Das bedeutet fr die vorliegende Arbeit, dass alle vorhandenen Tweets zur Generierung des Modells verwendet werden knnen.
Word2Vec stellt dabei keinen eignen Algorithmus dar. Es kann zwischen den zwei Architektur-Modellen "Continuous Bag of Words" (CBOW) und "Skip-gram" unterschieden werden. CBOW prognostiziert ein Wort auf Basis des Kontext. Skip-Gram arbeitet in die entgegengesetzte Richtung und prognostiziert den Kontext auf Basis eines Wortes. Angenommen der Satz "Wir lieben programmieren" dient als Beispiel, dann wrde CBOW anhand der Wrter "Wir" und "programmieren" das Wort "lieben" prognostizieren. Skip-gram prognostiziert anhand von "lieben" mit hoher Wahrscheinlichkeit die beiden anderen Wrter. Das Modell wird durch ein neuronalen Netz trainiert, indem die Modellgewichte in Form von Vektoren entsprechend angepasst werden. Fr einen tiefergehenden theoretischen Hintergrund wird an dieser Stelle auf die Arbeit der Entwickler Mikolov et al. (2013) verwiesen.

Fr die Implementierung von Word2Vec wird das Package gensim verwendet. Dafr mssen die Tweets zuerst in ein Format gebracht werden, welches fr das Training des Word2Vec Modells verwendet werden kann. 3.3 - Doc2Vec¶

Doc2Vec basiert auf dem bereits vorgestellt Modell Word2Vec. Der Unterschied liegt im Wesentlichen darin, dass bei Doc2Vec zustzlich zu den einzelnen Wortvektoren auch Vektoren fr das ganze Dokument trainiert werden. Dazu wird zustzlich zu den Wortvektoren ein eindeutiger Vektor fr das jeweilige Dokument bzw. in diesem Fall fr den ganzen Tweet erstellt. Dargestellt ist der zustzliche Vektor in der folgenden Abbildung durch die Farbe Orange und die Bezeichnung Paragragh id. Whrend sich der Ansatz "Distributed Bag of Words" (DBOW) dem Prinzip des Skip-Gram Modells bei Word2Vec zuordnen lsst, entspricht "Distributed Memory" (DM) dem CBOW Modell bei Word2Vec. DBOW betrachtet den Dokumentenvektor sowie einen zuflligen Wortvektor eines Dokuments und ermittelt Wahrscheinlichkeiten fr die weiteren Wrter des Dokuments. Das DM Modell ermittelt aus dem Kontext der umgebenden Wortvektoren und dem Dokumentenvektor ein fehlendes Wort. Dazu werden die Vektoren entweder verkettet (Distributed Memory Concatenated - DM-C) oder die Durchschnittswerte verwendet (Distributed Memory Mean - DM-M). Fr ein tieferes theoretisches Verstndnis wird an dieser Stelle auf die Arbeiten von Le und Mikolov (2014) sowie Rong (2016) verwiesen.

Die Implementierung von Doc2Vec erfolgt mittels des Packages gensim. Zum Training von Doc2Vec werden die Daten in dem speziellen Format TaggedDocument bentigt. Dieses enthlt jeden Tweet, zerlegt in seine einzelnen Worte. Dabei ist jedem Tweet eine eindeutige ID zugewiesen. Die Methode labelize_tweets( ) fhrt die Tweets aus dem Dataframe in diese Darstellung ber.

3.4 - Neuronale Netze¶

Knstliche neuronale Netze sind dem menschlichen Gehirn nachempfunden und bestehen in der Regel aus einem Input-Layer, einem Output-Layer und mehreren Hidden-Layer. Die Schichten enthalten eine bestimmte Anzahl an Neuronen, welche wiederum mit Neuronen der Folgeschicht verbunden sind. Diese Neuronenverbindungen enthalten Gewichtungen. Als Input dienen numerische Werte beispielsweise in Form eines Vektors. Das Inputsignal wird durch die Gewichtung und eine hinterlegte Aktivierungsfunktion verndert und als Outputsignal weitergegeben. Ein neuronales Netz lernt durch die Anpassung der im Netz enthaltenen Gewichte, mit dem Ziel den Fehler des Outputs mglichst zu verringern. Durch die numerische Abbildung von Texten durch beispielsweise Word2Vec oder Doc2Vec kann mittels neuronaler Netze eine Sentiment Analyse durchgefhrt werden. Im Folgenden dienen die mit Doc2Vec erstellten Vektoren der Tweets als Input fr die neuronalen Netze. Ziel ist es ein Modell zu traineren, welches eine mglichst hohe Accuracy ohne Over- oder Underfitting aufweist. Dazu werden mehrere Architekturen getestet und drei Modelle hier vorgestellt. Verwendet wird dazu die auf Tensorflow basierende Deep Learning Library Keras.

Die Modelle werden ber mehrere Epochen trainiert. Im Anschluss an jede Trainingsepoche wird die Accuracy auf dem Trainings- und dem Validierungsdatensatz ausgegeben. Zudem wird zu jedem Modell die Loss-Funktion und die Validation-Loss-Funktion ber die Trainingsepochen grafisch dargestellt. Dies dient der Kontrolle bezglich Overfitting.

    return vecs

3.5 - Convolutional Neural Networks¶

Convolutional Neural Networks (dt. in etwa: "faltende neuronale Netze") stellen eine andere Art von knstlichen neuronalen Netzen dar. CNNs haben ihren Ursprung in der Bildverarbeitung, werden aber vermehrt auch in anderen Bereichen eingesetzt. Ein CNN besteht im Wesentlichen aus dem Embedding Layer, mehreren Convolutional Layer, Aggregations-Schichten (Pooling Layer) und am Ende aus einer oder mehreren Schichten von "normalen" vollstndig verbundenen Neuronen bzw. Layer (Dense / Fully Connected Layer). Die Convolutional und Pooling Layer knnen mehrfach hintereinander platziert werden.

Um einem CNN die Tweets in numerischer Form bergeben zu knnen, wird eine einheitliche Formatierung der Daten bentigt. Das bedeutet jeder Tweet muss unabhngig von der enthaltenen Wortanzahl die gleiche Gre besitzen. Dazu wird den 100.000 hufigsten Wrter ein eindeutiger Index zugeordent. Diese Anzahl wurde gewhlt, da das Word2Vec Modell auf jeden Fall fr diese Wrter Vektoren erstellt hat und diese im zweiten Schritt mit dem CNN kombiniert werden. Um eine einheitliche Gre der Vektoren zu erreichen, werden Tweets, die weniger Wrter enthalten, mit der entsprechenden Anzahl an Nullen aufgefllt. Dem Wert Null ist kein Wort zugeordnet. Somit wird jeder Tweet durch eine Sequenz ganzzahliger positiver Werte reprsentiert. Diese Vektoren werden im Anschluss an das CNN bergeben. Der Embedding Layer des CNN enthlt fr alle Wrter eine Vektordarstellung der Lnge 200 (entspricht Lnge der Word2Vec Vektoren). Der Embedding Layer enthlt also 100.000 Vektoren. Diese Vektoren werden den jeweiligen Wrtern bzw. den entsprechenden Indizes der Wrter eines Tweets zugeordnet, sodass ein Tweet als Matrix dargestellt wird. Diese wird dann an den Convolutional Layer bergeben. Der Wert 0 ist dem Nullvektor zugeordnet.
Im ersten Ansatz werden die Vektoren des Embedding Layer vom Modell selbst gelernt. Die Vektoren werden dabei zufllig initialisiert und im Laufe des Trainings verbessert.
Im zweiten und dritten Ansatz werden die Vektoren des Word2Vec Modell fr den Embedding Layer vorgegeben. Jedes Wort wird also seinem Word2Vec Vektor zugeordnet. Im zweiten Ansatz werden diese Vektoren nicht aktualisiert, im dritten Ansatz hingegen trainiert das Modell diese Vektoren mit.

Aus Laufzeitgrnden werden im Folgenden lediglich CNNs mit einer einfachen Struktur betrachtet. Durch die Verwendung eines 1D Conv-Layer entspricht die Lnge der angewendeten Filter der Lnge der Zeilen bzw. Anzahl der Spalten der Tweet-Matrix, also 200. Somit wird der Filter jeweils auf die komplette Wortreprsentation (also nicht nur auf einen Teil davon) angewendet. Es werden 100 Filter mit der Dimension 2x200 angewendet. Dadurch werden je Filterschritt zwei Worte betrachtet.

  • cnn_01: CNN ohne vorgegebene Word2Vec Vektoren
  • cnn_02: CNN mit statischen Word2Vec Vektoren
  • cnn_03: CNN mit trainierbaren Word2Vec Vektoren

4.1 VADER Sentiment¶

Die Rckgabe der Methode getSentimentVader( ) ist ein Vektor, der den durschnittlichen Sentiment Score pro Tag, enthlt. Dieser kann zwischen -1 und 1 liegen und wird zur besseren Vergleichbarkeit mit den anderen Modellen auf [0, 1] normiert. Zustzlich wird die Anzahl der Tweets pro Tag zurckgegeben. Fr die Auswertung das Verhltnis von positiven zu negativen Tweets zu verwenden ist hier nicht sinnvoll. Da neutral klassifizierte Tweets als positive Tweets gezhlt werden, wrde dies zu einer starken Verzerrung des Gesamtsentiments fhren.

def getSentimentVADER(data):
    mean = []
    sentiment = []
    sent_date = []
    count = []
    #Erzeugung VADER Instanz
    sid = SentimentIntensityAnalyzer()
    for i in range(len(data.text)):
        if (i + 1) % 10000 == 0:
            print("Iteration: {} von {}".format(i + 1, len(data.text)))
        #In mean wird jeweils der Sentiments Score aller Tweets eines Tages gespeichert
        mean.append(sid.polarity_scores(data.text[i])['compound'])
        #Letzer Tweet
        if i == len(data) - 1:
            sentiment.append(((sum(mean) / len(mean) + 2) / 2)-0.5 )
            sent_date.append(data.date[i])
            count.append(len(mean)) #Anzahl der Tweets am letzen Tag
            break
        #Datum des vorherigen Tweets entspricht nicht dem Datum des nchsten Tweets
        if data.date[i] != data.date[i + 1]:
            sentiment.append(((sum(mean) / len(mean) + 2) / 2)-0.5 ) #Sentiment eines Tages wird gemittelt
            sent_date.append(data.date[i]) #Hinzufgen des Datums in count
            count.append(len(mean)) #Anzahl der Tweets ist Anzahl der Sentiment Scores
            mean = [] #Tweets eines Tages werden zurckgesetz
    print("Iteration: {} von {}".format(len(data.text), len(data.text)))
    #sentiment enthlt durchschnittlichen sentiment score pro Tag
    #sent_date enthlt Datum des jeweiligen Tags
    #count enthlt die Anzahl der Tweets pro Tag
    return sentiment, sent_date, count

def getSentimentNN(data):
    mean = []
    sentiment = []
    sent_date = []
    count = []
    pos_count = []
    for i in range(len(data)):
        if (i + 1) % 10000 == 0:
                print("Iteration: {} von {}".format(i + 1, len(data)))
        token_list = data.text[i].split(" ") #Tweet in einzelne Worte zerlegen
        #model.infer_vector(Liste von Worten) bestimmt den Doc2Vec Vektor des Tweets
        ivec = model.infer_vector(doc_words=token_list, steps=200, alpha=0.065)
        #predict_classes(Vektor) klassifiziert Ausgabe des NN direkt in "0" oder "1"
        mean.append(float(nn.predict_classes(np.array([ivec]))[0])) #Klasse wird zu mean hinzugefgt
        #Letzter Tweet
        if i == len(data) - 1:
            sentiment.append(sum(mean) / len(mean)) #Verhltnis von pos zu neg am letzen Tag
            sent_date.append(data.date[i]) #Datum letzter Tag
            count.append(len(mean)) #Anzahl Tweets letzter Tag
            pos_count.append(sum(mean)) #Anzahl positive Tweets letzer Tag
            break
        #Datum des nchsten Tweets entspricht nicht Datum des aktuellen Tweets
        if data.date[i] != data.date[i + 1]:
            sentiment.append(sum(mean) / len(mean)) #Verhltnis von pos zu neg des Tages
            sent_date.append(data.date[i]) #Datum des Tages
            count.append(len(mean)) #Anzahl der Tweets dieses Tages
            pos_count.append(sum(mean)) #Anzahl der positiven Tweets
            mean = [] #Tag wird zurckgesetzt
    print("Iteration: {} von {}".format(len(data.text), len(data.text)))
    return sentiment, sent_date, count, pos_count
